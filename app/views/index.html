Apply for ‘new style’ Employment and Support Allowance versions 0.8, 0.8b, 0.8c, 0.9

Created 3 new versions to test alongside version 8.

We aim to test:

  • order of questions
  • how we show progress through the application

These will be tested week commencing 29 April alongside existing version 8 prototype which has been iterated but not fully tested.

The versions are:

  • Version 8

    • - health questions first, linear form

  • Version 8b

    • - personal questions first, linear form

  • Version 8c

    • - health questions first with progress indicator at top of page, linear form

  • Version 9

    • - 'hub and spoke' application broken into sections linked to from a summary screen, based on GOV.UK task list pattern

Full version 8 notes - January 2019 to May 2019

Research showed:

  • Users still don’t understand that they have to do something once they submit
  • Self-employed users struggled with the 'Are you employed?' question if they aren’t in a contract at the moment
  • The claim date is still difficult to figure out, although the hint text did help
  • There is some jargon in the service that we need to describe - particularly fit note, SSP1, and claimant commitment
  • Permanent health insurance isn’t fully understood
  • Applicants very rarely have permanent health insurance - research on 438 claims showed 0 people with permanent health insurance
  • No users in our check across 438 claims had voluntary work and permitted work was also rare (4 out of 438)
  • 37 out of 438 claims checked had a pension

Changes:

  • Pension and permanent health insurance questions are now mostly optional, with a message at the top of each optional question to explain that anything they can give will save time later
  • Predictive type-ahead from health conditions removed as IRG needs updating because it contains offensive and outdated conditions
  • Information about the appointment is now inset on the start page to draw users’ eyes to it
  • Next steps on the confirmation page arranged as a numbered list
  • 'Add another condition' button added to the conditions screen - users can add up to 12 health conditions
  • Added a 'when it started' field to the conditions page as staff need it to understand the severity of the condition or if something has changed
  • Users whose sick pay ended a while ago are now asked 'When did your sick pay end?'
  • More hint text added to the claim date screen to start with - other ideas for how we might address this need below
  • Any jargon removed or described
  • New pension screens added to the question set - business requirement to ask gross and net amounts as well as deductions
  • Future hospital stays removed, and previous stays added - requested by Policy
  • Added ‘Are you expecting to go back to work?’ question
  • Permanent Health Insurnace question flow changed so that the user shouldn’t have to answer all the quesitons if they pay more than half the premiums
  • Changed consent screen as per Legal recommendation that users give consent for their doctors to share information, and updated the declaration with 'confirm' rather than 'agree' as the items in the declaration are obligations
  • Referring to ‘regular payments’ when asking about pensions
  • Re-ordered payment frequency options (pensions, insurance)
  • Removed previous hospital stays question – we are only interested if they’re in hospital now
  • Removed questions about expenses related to voluntary work
  • Updated ‘Done’ screen
  • Added a separate ‘Before you start’ screen
  • Add updated address fields to account for addresses that are outside the UK
  • Updated hint text on DS1500, terminally ill, and GP surgery screens after GDS content review

v0.7

Created: 07 Dec 2018

Research showed that users:

  • didn’t understand the application process
  • were confused by work screens as they didn’t cover the various ways people work
  • were anxious about giving fraudulent information, particularly dates

Significant changes:

  • GOV.UK pages and step by step navigation mocked up so we can see if the new step by step guide helps users understand the process
  • Start page slotted into GOV.UK pages to test route through to it
  • Check answers screen with dummy data entered to see if it helps with anxiety over fraud
  • Users can add multiple jobs
  • Company names carried through work and voluntary questions to reduce cognitive effort

Smaller changes:

  • Voluntary work questions moved to the start of the flow to reduce ambiguity between paid and voluntary work
  • Statutory sick pay questions amended to focus more on sick pay and whether its ended
  • Users asked for a date range rather than a date on the fit for work question to ease uncertainty
  • Additional options for pay frequency added (annually, quarterly, and every 6 months) to match JSAPS
  • New question added - Did you inherit your pension from [pension provider]?

v0.6

Created: 21 Nov 2018

Research showed:

  • users don’t understand that the application form is the first part of the process
  • some users found the Pension screens better to work through
  • users found the claim date hard to work out
  • having better content on the Start page did set expectations, but users needed to be prompted to read it

Significant changes:

  • Creating a flow of questions for capturing Pension payments, enabling users to add multiple providers

Smaller changes:

  • Updating content on the Start screen, improving the messaging to set better expectations for users
  • Content change to Condition question, adding 'injuries' to 'conditions' and 'illnesses'
  • Content changes to the Done screen, re-iterating that the application form is the first part of the claim process
